使用acme.sh脚本生成免费https证书

2022-02-23 #Docker #Linux
自动配置支持的域名服务商

https://github.com/acmesh-official/acme.sh/wiki/dnsapi

自动配置方式
export CF_Key="6fb7308ff0f5f59f31e67490764437cbae699"
export CF_Email="air@linzhe.com"

执行

acme.sh  --issue  --dns dns_cf -d hotelavish.com -d *.hotelavish.com --yes-I-know-dns-manual-mode-enough-go-ahead-please --force
1、安装

使用如下命令安装

curl  https://get.acme.sh | sh

会安装到 /~/.acme.sh/ 目录下，以后所有的配置默认也在这个目录下

设置 alias:

alias acme.sh=~/.acme.sh/acme.sh  # .bashrc or .zshrc or .config/fish/fish.
2、使用域名添加TXT记录方式设置

执行命令：

acme.sh  --issue  --dns -d example.com -d *.example.com \
	--yes-I-know-dns-manual-mode-enough-go-ahead-please

后面加上手动设置同意标识，泛域名设置需要带着根域名；

命令执行完成之后，会生成相应的解析记录，如下：

Multi domain='DNS:bitcat.cc,DNS:*.example.com'
Getting domain auth token for each domain
Getting webroot for domain='example.com'
Getting webroot for domain='*.example.com'
Add the following TXT record:
Domain: '_acme-challenge.example.com'
TXT value: '<ACME_CHALLENGE_STRING>'
Please be aware that you prepend _acme-challenge. before your domain
so the resulting subdomain will be: _acme-challenge.example.com
Please add the TXT records to the domains, and retry again.
Please add '--debug' or '--log' to check more details.
See: https://github.com/Neilpang/acme.sh/wiki/How-to-debug-acme.sh

设置两个域名可能会出现两条TXT记录，选择泛域名的即可，前往域名控制台添加TXT记录；

然后执行：

acme.sh  --renew  --dns -d example.com -d *.example.com \
	--yes-I-know-dns-manual-mode-enough-go-ahead-please

这时有可能会提示因为手动，需要再次验证，这时重新再走一遍步骤即可；

3、配置nginx

生成后的域名在/**~/.acme.sh/example.com(生成的域名)，**将证书复制出来，nginx配置如下：

listen 443 ssl;
server_name mark.example.com;

ssl_certificate /etc/letsencrypt/live/*.example.com/fullchain.cer;
ssl_certificate_key /etc/letsencrypt/live/*.example.com/*.example.com.key;
